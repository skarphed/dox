Users and Roles

In a skarphed-core there can be several users that are allowed to edit the contents and setup. How to manage them is being explained here.
One of the sub-objects of the skarphed-core in the object tree is a object called Users. If you expand it, you will see the users that are registered to that skarphed-core

The root User

On creation of a skarphed-core there is one special user. The so called root-user. The rootuser is the mightiest user on a skarphed-core due to the fact that no permission-checks apply to him. He is not meant for regular use like editing content. His purpose is to do administrative tasks and provide other users with permissions. The root-user is not deletable.

Creating a User

Right mouseclick on the Users-object in the object tree → Click on Create User.... Now a dialog should pop up that asks you for the name of the new user. Enter the name and press OK. You should now have a new user.

The User-objectpage

Permissions

Permissions allow users to do certain things, for example add a new role or install a module. In the Permissions-section you can adjust the permissions of a user. On the left side you can assign or revoke granular permissions from und to the user. Please note that you can only assign a permission to another user if you possess that permission yourself (the user you are logged in as) or if you are root. Be careful: you can revoke permissions from yourself and normally you won't want that.

Roles

On the right side you can assign or revoke roles from or to a user. Roles are basically accumulations of permissions and are being described more detailed in the section Roles. Assigning and revoking roles works similar to assigning permissions. Please note that, if you assigned a role to a user, the permissions that are included in this role, will not be displayed in the permissions-list on the right side anymore to keep the list simpler.

Password editing

If you have the permission to edit user's passwords or you look at your own user, you will see a Password-section. You can edit the users password here.

Deleting Users

You can delete a user by right mouseclicking on them in the object-tree and selecting Delete... in the context-menu. You will be asked to confirm the deletion. Confirm and your user will be deleted.

Roles

Another sub-object of skarphed-core in the object tree is the Roles-object. It enables you to create, edit and delete roles. Roles are basically lists of permissions that are thematically grouped together. So you can realize access-levels of persons more conveniently. Roles can be assinged to users exactly as permissions can be. When you expand the roles-object, you will see all roles that are currently known to the skarphed-core

Creating and editing a role

A role is created by right mouseclicking on the Roles-object in the object tree and selecting Create Role... in the context-menu. You will be asked for a name. Afterwards you will see your new Role in the object tree. Please double click on that role in the object tree to edit it. It's object page looks kind of similar to the one of ther User. You can simply add and revoke permissons from or to the role by clicking the checkboxes left to the corresponding permission in the list.

Deleting a role

Right mouseclick on the role in the object tree and select Delete... in the context-menu. You will be asked to confirm to decision. After you confirmed, the concerned role will be deleted.
