Databases

Step 1: Installing the server

Skarphed uses the Firebird RDBMS as it's database management system. Therefore you have to set up at least one Firebird-server in your setup. It is up to you which operating system you run your Firebird-server on, it just has to offer a SSH-Server at TCP-port 22, the database connection at TCP-port 3050 (which is firebirds default port) and must be a running a Firebird version 2.5 or greater. On Debian (tested on squeeze) for example you install a Firebird-server simply by executing these commands:
user@skarphed-db:~$ sudo apt-get install firebird2.5-super
user@skarphed-db:~$ sudo dpkg-reconfigure firebird2.5-super
user@skarphed-db:~$ sudo /etc/init.d/firebird2.5-super restart
The second command will ask if the Firebird-server should be started by default. Answer this question with yes. Further it will ask you for a SYSDBA-password. SYSDBA is the equivalent of a root-user in UNIX-environments. Choose a strong password and keep it until the next step.

Step 2: Register the database in skarphed-admin

If you haven't already done it, please register the machine you just installed Firebird on as a server in skarphed-admin. Now you do a right mouseclick on this server and click Register Database. You will now see a Dialog in the working area that asks you for a Firebird-database-user and a password. Please enter "SYSDBA" and the password you chose for the User in the last step. Finish by clicking OK.
