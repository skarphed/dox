FAQ

What is skarphed?

Skarphed is a software suite that is meant to set up, manage, maintain or tear down websites for you. Therefore skarphed consists of three distinct programs, which are skarphed-admin (the administration GUI), skarphed-core (a CMS that runs on the webservers) and skarphed-repo (a module repository that supplies modular extensions for skarphed-cores)

Why do you write skarphed?

Because i have been told to use CMS for website development, but I was not able to find any that suits me.

Which problems does skarphed solve that haven't been solved yet by similar systems?

  • Code Signing:

    When you add modular functionality to most CMS-systems, you are usually bound to use code that you find at the page of the developer and trust all parts of the delivery chain, that it has not been contaminated with malicious code in the meantime. Skarphed however uses public-key cryptosystems to sign all the code that is being supplied from third party members to components of skarphed and comes with it's own code distribution infrastructure, the skarphed-repo. Signing happens everytime when skarphed-components exchange code.

  • Native GUI:

    The usual mainstream CMS uses WebGUIs to allow the website adminstrator to edit the content. As a matter of fact, I don't like WebGUIs due to their lack of responsiveness and want to provide an alternative that uses a native GUI.

  • Password uniqueness:

    If you have a bunch of websites to maintain, it is hard to use strong admin-passwords for every page and keep them in your head. As a result, many users tend to either write their passwords down in a .txt-file or simply choose easily rememberable and therefore easily guessable passwords. Skarphed solves that problems by letting the skarphed-admin act as a password manager. The user simply has to remember his own profile password.

  • Maximized code-reusability:

    Write a Module (e.g. a Forum) once, deploy it on every other skarphed-core you need it, too. Simply via drag and drop. This is how i think it is meant to be. This is why i coded it this way.

  • Setup routine:

    With most of the CMS it's always the same routine. Set up a server, install a webserver, copy files, install a database, create a database schema, import the template database schema of the CMS, and so on... Skarphed does that for you on one click. The only thing you have to do is to set up a server and tell your skarphed-admin it's SSH-credentials.

This list is not comprehensive. Other innovative stuff may follow. There is so much to do better.

Which kind of people is your intended audience?

Generally everyone who read the "solved problems"-list above and thinks "hey cool, that makes sense" and wants to create and manage some websites. Everyone who has to manage many websites anyway.

What technology is skarphed based on?

Skarphed uses Python 2 for nearly everything. The GUI of skarphed-admin is written in pygtk and so are the GUIs of skarphed's modules. the skarphed-core runs on WSGI, the python default webserver gateway interface.

On what systems do the skarphed components run?

The skarphed-admin will work on GNU/Linux, Microsoft Windows and Apple MacOS. Currently it only works fully under GNU/Linux but there are proof of concepts that ports to Windows and MacOS are possible. And we will make them reality. We don't know yet which versions of Windows and MacOS will be supported. Basically, skarphed-admin works everywhere, where Python 2, a Gtk-Runtime and a c-compiler is available.
The skarphed-core runs on GNU/Linux with every Webserver that supports WSGI. So does the skarphed-repo.

Which DBMS does skarphed use?

Skarphed uses Firebird.
