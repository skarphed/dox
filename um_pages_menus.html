Pages and Menus

Pages contain placeholders for widgets. They define the visual layout of a page in your website. Usually websites have only one page, the layout of your page, whereas more sophisticated webapplications eventually need more pages. In the object-tree you see Sites (how pages are currently still called inside skarphed-admin due to legacy code) as a sub-object of your skarphed-core. It will be filled with content as soon as you installed a template. Pages have no object-page due to the fact that there is nothing to configure.

Spaces

Each page containts placeholders where you can place widgets in (e.g. guestbook, blog, or a simple html-page). A classical pagelayout would be something like: a space for the menu, a content space, a footer space and a sidebar. You cannot see these spaces directly as sub-object of the page in the object-tree, but you can use them when you design views and menus.

Menus

Menus represent the logical structure of actions a user can take to navigate in your website. Menus hold menuitems. Menuitems are the links that will appear on the page e.g. 'Blog', 'Home' or similar stuff. Each menuitem holds a list of actions. Actions can be three things:

  • Call URL

    An action of this type will simply call an URL like http://www.skarphed.org. You use it to create external links.

  • Place widget into space

    This type of action will load a widget that you specified into a space of the page this menu belongs to.

  • Open View

    This type of action will load another view that is known to this skarphed-core

Obviously, using multiple actions in one menuitem only make sense if they all are "Place widget into space"-actions.
Menuitems can be nested virtually arbitrarily deep. Nested menuitems make sense if you e.g. want to design a tree-style menu.
Please make sure that you understand, that this only represents the logical structure of a menu. How the actual menu looks rendered on a page is determined by a menu-widget that takes this structure of data and renders it on the page.

Creating a Menu

Right mouseclick on the page you want to add your menu to and select Create Menu.... You now should be able to expand the page and see your new menu.

Editing Menus

You can open the menu's object-page by (you guessed it) doubleclicking in it in the object-tree. You can change the name of the menu at the entryfield at the top. At the left side you see the Menu tree. It bears only one element, which is the menu itself. You can add menuitems to it by selecting the element you want to add the new items to (in our case the menu itself) and pressing the Add-Button in the Menu tree's toolbar. To rename your menuitems please select the menuitem and then click on it's name in the list. It will change to a entryfield that you can edit the menuitems name in. If you are done editing simply press enter and the name will be changed.
At the right side you see the actionlist-editor. Everytime you select a menuitem in the Menu tree all actions of this menuitem will be displayed here. If you want to add a new action, you can use the Add-Button on the bottom of the actionlist-editor. It will spawn in an collapsed state. You can decollapse and edit it by clicking the expander-arrow. Edit the action and save it via it's Save-button.
