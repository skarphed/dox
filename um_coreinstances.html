Instances of skarphed-core

The skarphed-core is the component of skarphed that runs on the webserver. Each instance of skarphed-core represents one website or webapplication. You can run multiple skarphed-cores on one server.

Installing a new skarphed-core

To install a skarphed-core-instance on a server, the following prerequisites must be met:

  • There must be a database in your infrastructure
  • There must be a repository in your infrastructure
Optionally it would make sense to have a DNS-name prepared for the new skarphed-core. For example myskarphedpage.com.
When you verified that you meet all prerequisites, please choose the server you want to install the new core on and right mouseclick on it. Choose Create Instance... in the contextmenu. A Dialog will be opened that wants you to enter the following data:
  • Server

    The server the new instance will be installed on. This should be automatically set so the server your right-clicked on.

  • New Instance Name

    Enter an appropriate name for your instance here. For example, if you create a website for the FooBar-Company, name it foobar, foobarsite or FooBarsSite or something similar.

  • Repository

    The skarphed-repo that this skarphed-core will use to get modules and templates. The dropdownbox knows every skarphed-repo that is registered to your profile.

  • Database

    Please choose either Create new Database and select the Databaseserver that the schema for the new skarphed-core should be created on or Use existing Schema to use a schema that already exists. The dropdownboxes know every schema or database registered to your profile.

  • Target-OS

    The Target is a combination of an operating system and a webserver that the new instance will run on. Currently there is only a combination of Debian6 and Apache2. Others will follow. Feel free to implement your own and submit them to the project!

  • Listen-IP

    The IP this skarphed-core will listen on

  • Listen-Port

    The TCP-Port this skarphed-core will listen on

  • ServerName

    The DNS that this server will answer. This is necessary if you want to consolidate many skarphed-cores on one server. For example if you have two domains, example1.org and example2.org, the webserver can distinguish via this directive which skarphed-core the HTTP-request should be redirected to.

  • ServerAlias

    The alias is an alternative DNS-name for this skarphed-core. For example if you want the skarphed-core to listen on example1.org and on www.example1.org simultaneously. Or if you want this skarphed-core to listen on example1.org and every subdomain of example1.org (use *.example1.org here).

When you finished entering all relevant data, click OK and wait until the progress bar reached 100%. This may probably take a longer time, especially when you havent had installed a skarphed-core on this server before or your server has a slow internet connection. The internet connection is required to install dependencies of skarphed-core on the server.
